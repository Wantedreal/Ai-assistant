## Project Overview
Desktop application
the User enters a housing size and energy\current targets.
The app calculates the battery pack Series\Parallel configuration and returns ACCEPT\REJECT.

Backend in the \backend : FastAPI + SQLITE + Python, running on localhost:8000.
Frontend in the \frontend: React  + Vite loaded inside Electron.

---
### How to run (Development):
```powershell
# Terminal 1: Start backend
cd backend
.\venv\Scripts\Activate.ps1
uvicorn app.main:app --reload --host 0.0.0.0 --port 8000

# Terminal 2: Start Vite dev server
cd frontend
npm run dev

# Terminal 3: Launch Electron
cd frontend
$env:ELECTRON_DEV='true'
npm run dev:electron
```

### How to build (Production):
```powershell
# 1. Build backend binary with PyInstaller
cd backend
.\venv\Scripts\Activate.ps1
pyinstaller --onedir --name backend --add-data "data;data" --paths . app/main.py

# 2. Build frontend + package Electron installer
cd frontend
npm run electron:build
# Output: frontend/release/Battery Pack Designer Setup 1.0.0.exe
```

- Do not overengineer. Do not program defensively.

## Debugging Protocol 
When something breaks, follow this order :
1. Reproduce consistently
2. Investigate + form a hypothesis
3. Demonstrate the root cause
4. Fix and prove the fix works
5. Document lessons learnes in 'AGENTS.md'
> Don't guess. Don't jump to conclusions. Don't apply workarounds.
> Try one test at a time. Be methodical
